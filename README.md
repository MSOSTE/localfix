# LocalFix

LocalFix is a Django web application for reporting neighborhood issues.

Users can create reports, add photos, filter reports, and follow their status. Moderators can manage all reports and change report statuses.

## Features

- User registration, login and logout
- Cookie/session based authentication
- Two application roles: user and moderator
- PostgreSQL database
- Latvian and English language switching
- Issue report CRUD
- Report categories
- Photo upload and photo deletion
- Report status filtering
- "My reports" filter
- Pagination after 10 reports
- Django admin panel for managing users, profiles, categories and reports

## Roles

- User:
  - can create reports
  - can edit and delete only own reports
  - can view all reports

- Moderator:
  - can edit and delete all reports
  - can change report status
  - can view all reports

Moderator role is assigned in the Django admin panel:

```text
Accounts -> Profiles -> role = moderator
```

## Run locally

Install dependencies:

```bash
pip install -r requirements.txt
```

Apply migrations:

```bash
python manage.py migrate
```

Start the server:

```bash
python manage.py runserver
```

Open:

```text
http://127.0.0.1:8000/lv/
```

## Database

The project uses PostgreSQL.

Create a database named `localfix`, then configure environment variables in `.env`:

```text
DJANGO_SECRET_KEY=change-me
DJANGO_DEBUG=1
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost
POSTGRES_DB=localfix
POSTGRES_USER=localfix
POSTGRES_PASSWORD=localfix
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_CONNECT_TIMEOUT=5
```

Use `.env.example` as a template. Do not commit `.env`.

## Admin Panel

Create an admin user:

```bash
python manage.py createsuperuser
```

Open:

```text
http://127.0.0.1:8000/lv/admin/
```

The admin panel is generated by Django and configured in `admin.py` files.
